===============================================================================
6. FAIRNESS ASSESSMENT
===============================================================================

This experiment has an inherent methodological asymmetry that must be
disclosed: the comprehensive ground truth was seeded primarily from
SonarQube output (482/497 = 97%), making SonarQube's near-perfect
score (97.8%) structurally guaranteed rather than independently earned.

WHAT THIS EXPERIMENT VALIDLY SHOWS:

  1. COMPLEMENTARY STRENGTHS. The two tools are strong in different areas:
     - SonarQube excels at ENUMERATION: systematically flagging every
       instance of a known pattern (all raw types, all missing @Override,
       all complexity violations). It is exhaustive and consistent.
     - The AI excels at COMPREHENSION: understanding semantic meaning,
       detecting logic errors that require reasoning about program
       behavior (inverted conditions, integer overflow, off-by-one errors).

  2. CATEGORY-SPECIFIC PERFORMANCE (adjusted for GT bias):
     - syntax_style:     SonarQube WINS decisively (100% vs 22.8%)
     - quality_warning:  SonarQube WINS decisively (100% vs 14.5%)
     - code_smell:       SonarQube WINS (98.1% vs 47.2%)
     - security:         SonarQube WINS on pattern-based nullity (100% vs 6.7%)
                         but these are all SQ-sourced null-deref warnings
     - logic_error:      COMPARABLE (61.5% vs 57.7%), and the AI found
                         8 logic bugs that SonarQube completely missed

  3. FALSE POSITIVE INTERPRETATION:
     - SonarQube's 0 FPs is an artifact of the GT construction method.
     - The AI's 82 "FPs" include many genuine issues (null-safety gaps,
       integer overflow risks, contract violations) that a human auditor
       would likely accept. A manual review of the AI's 82 FPs would
       likely reclassify 30-50% as true positives.

  4. COMBINED VALUE:
     - The union of both tools covers 497 + ~40 valid AI-unique findings
       ≈ 537 total issues, more than either tool alone.
     - SonarQube provides breadth (systematic pattern coverage).
     - The AI provides depth (semantic understanding of logic flows).

RECOMMENDATION FOR THE THESIS:

Present BOTH experiments. The first experiment (Defects4J-only GT)
shows AI superiority on semantic bugs (82.5% vs 5.0%). This second
experiment shows SonarQube superiority on pattern-based quality issues
(97.8% vs 22.1%). Together they demonstrate that static analysis and
AI review are COMPLEMENTARY, not competing — each catches defects the
other misses. The strongest code review process uses both.

A fair single-number comparison across ALL defect types would require
a ground truth built independently of both tools (e.g., by human
expert audit), which is outside the scope of this experiment.
